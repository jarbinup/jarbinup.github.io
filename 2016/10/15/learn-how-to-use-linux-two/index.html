Linux 基础入门（二）

发表于 2016-10-15   |   分类于 工具   |  

本篇包括三部分：

目录结构及文件基本操作
环境变量与文件查找
文件打包与解压缩

目录结构

Linux 中一切皆文件。Linux 是以树形目录结构来构建整个系统的。本质上无论是目录结构还是操作系统内核都是存储在磁盘上的，但逻辑上来说 Linux 的磁盘是挂载在目录上的。每个目录不仅能使用本地磁盘分区的文件系统，也可以使用网络上的文件系统。

Linux 下的目录结构是按照 FHS( Filesystem Hierarchy Standard ) 标准定义的。
FHS 定义了两层规范，第一层 / 下面的各个目录应该要放什么文件数据，例如 /etc 应该要放置设置文件，/bin 与 /sbin 则应该要放置可执行文件等等。
第二层则是针对 /usr 及 /var 这两个目录的子目录来定义。例如 /var/log 放置系统登录文件,/usr/share 放置共享数据等等。

目录路径

cd 命令切换目录， . 表示当前目录, .. 表示上一层目录 ( 目录本身也是文件)。
输入 ls 默认是隐藏的， ls -a 查看隐藏文件。
pwd 获取当前路径
cd - 上一次所在目录
cd ~ 当前用户家(主)目录

绝对路径，以根目录 / 为起点进入所要到的目录，像这样 /usr/local/bin
相对路径，以当前目录为起点，以当前目录为起点 ., 像这样 usr/local/bin(这里假设但前目录为根目录)
以当前用户的家目录为起点，两种方式进入特定路仅

基本文件操作

touch filename 创建文件
mkdir testdir 创建目录
mkdir -p father/son/grandson, -p 参数，同时创建父目录。
cp file path 复制一个文件到特定的目录中。
cp -r dir path 复制一个文件夹到特定目录
-r 或 -R 表示递归复制
rm file 删除一个文件
rm -f file 表示强制删除
rm -r dir 删除一个目录，也要加上 -r 参数。
mv file path将特定文件移动(剪切)到特定的目录。
mv filename newfilename 文件重命名
rename 's/\.txt/\.c/' *.txt 批量重命名

cat filename 查看文件
cat -n filename -n 参数显示行号
less more 命令分页查看文件
head tail 查看头/尾行，默认显示 10 行

file filename 查看文件类型, Linux 中不以文件后缀判断文件类型.

环境变量

环境变量就是作用域比自定义变量要大。简单理解，若在子进程中有效，则为环境变量，否则不是。

为了和普通变量区分，通常我们习惯将环境变量名设为大写。

搜索文件

whereis who
简单快速，只能搜索二进制文件(-b),man帮助文件(-m)和源代码文件(-s)
locate /etc/sh
快而全
which
小而精，只从 PATH 环境变量指定的路径中去搜索命令
find
精而细

文件打包和解压缩

常见的方式有 zip, rar, tar 三种 。
以 tar 为例，tar 本身只打包不压缩
tar -cf test.tar ~ 创建一个包
tar -xf test.tar -C tardir 解一个包到 tardir 目录中

tar -czf test.tar.gz ~ 创建一个包,并用 gzip 来压缩文件
tar -xzf test.tar.gz -C tardir 解一个包到 tardir 目录中

( 完 )
