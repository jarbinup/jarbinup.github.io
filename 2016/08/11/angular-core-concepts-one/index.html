ng 核心概念梳理 ( 一 )

发表于 2016-08-11   |   分类于 fed   |  

数据绑定(DB)

  1. module 层的数据和 view 层的数据实现自动同步。
  2. view 层是 module 层的一个 映射 ( projection ), 反映视图层的改变
  3. 双向数据绑定

Controllers

  1. ng 中，controller 就是一个 js 构造器函数,用来增加 Angular Scope
  2. 当一个 Controller 通过 ng-controller 指令关联到一个 DOM 节点上。Angular 会通过这个Controler构造函数实例化一个新的controller 对象。同时，会产生一个新的子作用域 child scope。在 controller 中通过传入的 $scope参数，访问这个参数。
  3. controller 主要用来 建立 $scope 对象的初始化状态，为 $scope 对象添加行为(业务逻辑)。
  4. 不要在 controller 中：
    a. 操作DOM, 用指令代替
    b. 格式输入, 用 angular form controls 代替
    c. 过滤输出, 用 angular filters 代替
    d. 共用/共享 代码或状态，angular service 代替
    e. 管理组件的生命周期
  5. controller 中只应该包含对单一视图的必须业务逻辑。保持业务逻辑轻量最常见的方法，将不是业务逻辑的功能封装在services中。
  6. 可以通过ng-controller 和 $route service 隐式的将 Controller 和 scope object 关联起来
  7. 作用域继承( Scope Inheritance ) child scope 可以继承 parent 的属性，在自己的 scope 中，可以重写继承的属性。

Services

  1. 通过依赖注入的方式使用 service
  2. 创建 services, 通过Module API注册 service 在 module 上，也可以通过 $provide 例如
var myModule = angular.module( 'myModule', []);
myModule.factory('serviceId', function() {
    var newService;
    //...
    return newSerive;
});

angular.module('myModule', [])
    .config([ '$provide', function( $provide ) {
    var newService;
    //...
    return newSerive;
    }]);
